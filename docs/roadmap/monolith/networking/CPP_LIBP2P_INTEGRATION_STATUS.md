## Resolved ABI Conflicts

### Dependency version matrix

| Dependency | Teleno node (Hunter a20151e) | cpp-libp2p source / aux deps | Resolution |
|---|---|---|---|
| Boost | 1.83.0 | aux Hunter headers | Use Koinos Boost first and a Beast shim for the missing header-only component config |
| Protobuf | 3.17.3 (koinos fork) | cpp-libp2p previously used 3.19.4-p0 | Rebuild cpp-libp2p against Koinos Protobuf and isolate aux include dirs |
| OpenSSL | OpenSSL 3.0.12 | cpp-libp2p previously used BoringSSL | Patch cpp-libp2p to use OpenSSL and disable QUIC/lsquic |

### Protobuf ABI break (3.17.3 â†’ 3.19.4)
Previous missing symbols when linking:
- `google::protobuf::internal::RepeatedPtrFieldBase::DestroyProtos()`
- `google::protobuf::Message::MaybeComputeUnknownFieldsSize()`
- Several other internal methods added in 3.19

### OpenSSL vs BoringSSL
- **lsquic** (QUIC transport) uses `EVP_AEAD_*` â€” BoringSSL-only
- **chachapoly_impl.cpp** uses `EVP_AEAD_*`, `bssl::ScopedEVP_AEAD_CTX` â€” BoringSSL-only
- **gRPC** (Teleno node) uses `SSL_ctrl`, `SSL_get1_peer_certificate` â€” standard OpenSSL-only
- Cannot link both BoringSSL and OpenSSL in same binary

5. **`src/crypto/chachapoly/chachapoly_impl.cpp`** â€” Rewritten to use standard OpenSSL 3.x EVP API:
   - `EVP_chacha20_poly1305()` instead of `EVP_aead_chacha20_poly1305()`
   - `EVP_EncryptInit_ex`/`EVP_DecryptInit_ex` instead of `EVP_AEAD_CTX_seal`/`open`
   - `EVP_CTRL_AEAD_GET_TAG`/`SET_TAG` for authentication tag handling
   - Removed `bssl::ScopedEVP_AEAD_CTX`, using manual `EVP_CIPHER_CTX_new`/`free`

6. **`include/libp2p/crypto/chachapoly/chachapoly_impl.hpp`** â€” Removed `EVP_AEAD_CTX*` and `EVP_AEAD*` members

### Build approach used
Instead of fighting Hunter version overrides, cpp-libp2p is built in `PACKAGE_MANAGER=vcpkg` mode (which just uses `find_package` for everything) with `CMAKE_PREFIX_PATH` pointing at:
1. Teleno node's Hunter install (for Protobuf, OpenSSL, Boost, ZLIB, c-ares)
2. cpp-libp2p's Hunter install (for soralog, tsl_hat_trie, Boost.DI, fmt, yaml-cpp, libsecp256k1, qtils)

Aux headers from the second prefix are copied into `node/teleno-node/.deps/cpp-libp2p-thirdparty-include/` so the old Protobuf/OpenSSL headers are not visible to the Teleno node build.
