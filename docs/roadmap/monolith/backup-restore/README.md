# Teleno Native Backup Documentation

This directory tracks the native `teleno_node` backup and restore work.

## Active Documents

- `NATIVE_BACKUP_CURRENT_IMPLEMENTATION.md` describes the implementation that is currently in the branch.
- `NATIVE_BACKUP_REMAINING_WORK_PLAN.md` is the active backlog and validation plan for the remaining work.
- `PUBLIC_BOOTSTRAP_RESTORE_PLAN.md` describes the implemented public read-only restore path for first-time testnet and ProdNet installs, including published snapshot and validation evidence.
- `PUBLIC_BOOTSTRAP_RESTORE_REMAINING_WORK_PLAN.md` tracks release-hardening work after the testnet and ProdNet implementation.
- `PRODNET_PUBLIC_BOOTSTRAP_PUBLICATION_PLAN.md` defines the ProdNet public bootstrap publication workflow and current release-governance follow-ups.
- `PRODNET_PUBLIC_BOOTSTRAP_VALIDATION_20260621.md` records the ProdNet public-bootstrap list result and operator production-node validation signoff.
- `NATIVE_LIBSSH_TESTNET_VALIDATION_20260614.md` records the latest live-testnet and restricted-SFTP validation evidence.
- `MONOLITH_BACKUP_RESTORE_REPORT.md` is historical evidence for the earlier monolith restore path.
